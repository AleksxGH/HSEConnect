# ВЫШКоннект

Корпоративная социальная платформа для студентов, преподавателей и сотрудников НИУ ВШЭ.

## Демо

**Сайт:** https://hseconnect-wf7e.onrender.com


> Важно: проект размещён на бесплатном тарифе Render.
> Если сайт долго не открывается, это нормально — сервер может находиться в спящем режиме. Обычно ему требуется 1–2 минуты для запуска после первого запроса.

---

## О проекте

ВЫШКоннект — социальная платформа для университетского сообщества, позволяющая студентам и сотрудникам:

- создавать и редактировать мероприятия;
- находить новых знакомых по интересам;
- подписываться на пользователей;
- добавлять друзей;
- обмениваться сообщениями в реальном времени;
- получать уведомления;
- управлять собственным профилем;
- получать рекомендации пользователей.

Проект разработан в рамках группового проекта по курсу «Конструирование баз данных».

## Основные возможности

### Пользователи

- Регистрация и авторизация
- Заполнение анкеты
- Редактирование профиля
- Загрузка аватаров
- Поиск пользователей
- Блокировка пользователей

### Социальные связи

- Подписки
- Подписчики
- Друзья
- Заявки в друзья
- Управление отношениями между пользователями

### События

- Создание мероприятий
- Редактирование мероприятий
- Удаление мероприятий
- Регистрация на мероприятия
- Приглашение пользователей
- Просмотр участников
- Загрузка фотографий мероприятий

### Сообщения

- Личные чаты
- Обмен файлами и изображениями
- Отметка сообщений как прочитанных
- Подсчёт непрочитанных сообщений
- WebSocket-обновления в реальном времени

### Уведомления

- Новые подписчики
- Заявки в друзья
- Приглашения на мероприятия
- Счётчик непрочитанных уведомлений
- Обновления через WebSocket

## База данных

Проект использует PostgreSQL.

Особенности схемы:

- нормализация до 3НФ/BCNF;
- более 30 сущностей;
- составные ключи для связей многие-ко-многим;
- ограничения целостности;
- индексы для оптимизации запросов;
- представления (Views);
- транзакции для сложных бизнес-операций.

## Интерфейс

В системе реализованы:

- регистрация;
- авторизация;
- главная страница событий;
- профиль пользователя;

## Особенности реализации

- Реальное время через WebSocket
- Поддержка непрочитанных сообщений
- Защита от дублирования подписок и регистраций
